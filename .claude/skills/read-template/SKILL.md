# Skill: read-template（模板读取/提取）

## 触发时机

Phase 1：用户提供了模板文件（.docx 或 .md 格式）后。

## 输入

- `template_path`: 模板文件的绝对路径
- `mode`: `pure-template`（纯模板文件）或 `extract-from-article`（从用模板写的文章中提取模板）
- `output_name`: 模板名称，用于命名产出文件

## 执行步骤

### Step 1：读取源文件

**如果是 .docx 文件**：
```bash
pandoc "<template_path>" -o templates/.tmp-read.md --track-changes=all
```

**如果是 .md 文件**：
直接读取。

### Step 2：分析并记录（根据 mode）

#### mode = `pure-template`（纯模板）

记录以下所有要素：

**页面格式**：
- 页面尺寸（A4 / US Letter）
- 页边距
- 页眉/页脚内容（页码格式、公司名等）
- 是否包含封面页
- 是否包含目录

**标题层级体系**：
- 各级标题的字体、字号、加粗、颜色
- H1/H2/H3 的数量和结构模式
- 标题编号方式（如 1. / 1.1 / 1.1.1）

**正文样式**：
- 正文字体、字号
- 段落间距（段前/段后）
- 行距
- 首行缩进

**表格样式**：
- 表头样式
- 边框样式
- 单元格边距

**列表样式**：
- bullet 还是 numbered
- 缩进层级

#### mode = `extract-from-article`（从文章中提取）

除了以上格式要素，还需识别：

**结构模式**：
- 文章的整体章节组织方式
- 哪些章节是必有的，哪些是可选的
- 章节间的逻辑关系

**措辞风格**：
- 语气（正式/半正式/口语）
- 人称（第一人称/第三人称）
- 常见句式（如"本文旨在..."、"综上所述..."）

**内容模式**：
- 每个章节通常写什么（摘要/正文/结论的分配）
- 引用风格
- 图表使用方式

### Step 3：写入模板文档

将分析结果写入 `templates/<output_name>-template.md`，使用以下结构：

```markdown
# 模板：<output_name>
提取自：<源文件路径>
提取时间：<ISO 8601>

## 格式规范
- 页面：<A4 / US Letter>
- 字体：标题 <Xpt> / 正文 <Xpt>
- 标题层级：<H1 → H2 → H3>
- 段落间距：段前 <X> / 段后 <X>
- 行距：<X倍>

## 结构规范
1. <封面页：有/无>
2. <目录：有/无>
3. 正文结构：
   - <章节1>：<章节名>（<H1/H2/H3>）<必有/可选>
   - <章节2>：<章节名>（<H1/H2/H3>）<必有/可选>
   ...

## 样式要求
- 段落：<首行缩进/无缩进>
- 列表：<bullet / numbered>
- 表格样式：<边框线型 / 表头底色>
- 页眉：<内容>
- 页脚：<内容>

## 措辞规范
- 语气：<正式/半正式/口语>
- 人称：<第一人称/第三人称>
- 常见句式示例：
  - <示例1>
  - <示例2>

## 关键约束
- 必须包含：<...>
- 不能包含：<...>
- 字数/篇幅：<每章约X段，全文约X字>
```

### Step 4：通知主流程

模板文档生成后，输出摘要信息给主流程。

## 禁止行为

- 不要修改用户提供的原始模板文件
- 不要编造任何格式信息（不确定的标注"未检测到"）
- 不要在提取模板的同时尝试写文章
- 不要跳过任何样式要素的记录

## 硬停止 (Hard Stop)

模板提取完成。产出文件：`templates/<output_name>-template.md`。

**请主流程启动 template-checker subagent 审核此模板文档。**

**严禁自行进入下一阶段。严禁调用 write-article。**
必须等待 template-checker subagent 审核通过后，由主流程决定是否进入 Phase 2。
