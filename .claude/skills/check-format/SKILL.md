# Skill: check-format（文章格式检查）

## 核心策略

检查的是 Markdown 草稿（内容+结构），而非 .docx XML。因为 .docx 由 pandoc + .dotx 生成，格式正确性由 pandoc 和模板保证。本技能专注于**结构合规性**和**内容质量**。

## 输入

- `template`: 模板文档路径（`templates/<name>-template.md`）
- `draft`: 文章 Markdown 草稿路径（`output/<name>-draft.md`）
- `docx`: 生成的 .docx 路径（可选，用于视觉对比）

## 执行步骤

### Step 1：读取对照基准

1. 完整读取模板文档
2. 提取检查清单：
   - **必须项**：模板「关键约束」中的"必须包含"列表
   - **禁止项**：模板「关键约束」中的"不能包含"列表
   - **结构标准**：模板「结构规范」中的章节列表（名称、层级、必有/可选）
   - **样式标准**：模板「样式要求」和「格式规范」

### Step 2：结构检查

逐项比对文章的标题层级和章节结构：

1. 提取文章中所有标题（`#`、`##`、`###`...），形成层级树
2. 与模板的章节结构对照：
   - 必须项是否齐全
   - 标题层级是否与模板一致
   - 章节顺序是否正确
   - 是否有模板未定义的额外章节（不禁止，但标注）
3. 记录所有不匹配项

### Step 3：内容质量检查

1. **章节充实度**：每章是否 >= 3 段实质内容？是否有"一句话带过"的章节？
2. **空话检测**：搜索常见的 LLM 水话模式：
   - "在当今时代/社会"
   - "随着...的发展/深入"
   - "众所周知"
   - "不可否认"
   - "在一定程度上"
   - "综上所述"（过度使用）
3. **资料引用**：是否能在资料中找到对应支撑？标注"无法验证的断言"
4. **数据事实**：是否有看起来编造的数据（如精确百分比无出处）？

### Step 4：样式一致性检查

1. **列表标记**：bullet 和 numbered 使用是否一致
2. **表格**：是否有表头、是否有空单元格
3. **标题风格**：同级标题的写法是否一致（全是陈述句/全是问句/混用）
4. **术语一致性**：同一概念是否用同一词表达

### Step 5：生成检查报告

写入 `reports/<output_name>-check-<timestamp>.md`：

```markdown
# 文章检查报告
检查时间：<ISO 8601>
模板：<template_path>
草稿：<draft_path>

## 结构检查
| 状态 | 项目 | 说明 |
|------|------|------|
| ✅ | 标题层级 | H1→H2→H3 与模板一致 |
| ❌ | 参考文献 | 模板要求但文章中不存在 |
| ⚠️ | 附录 | 模板中可选，当前未包含 |

## 内容质量
### 必修复
- ❌ 第X章只有 2 段，内容不充实
- ❌ 第Y章发现编造数据："XXX"无出处

### 建议修复
- ⚠️ 全文发现 N 处空话（附位置）
- ⚠️ 第Z段"XXX"断言无法在资料中验证

## 样式一致性
- ✅ 列表格式正确
- ⚠️ 第X章表格缺少表头
- ⚠️ H3 标题风格不统一（问句 vs 陈述句混用）

## 裁决
必须修复项：X 个
建议修复项：Y 个

- [ ] 通过（必须修复项 = 0）
- [x] 不通过，需要修改
```

## 禁止行为

- 不要直接修改文章
- 不要提供具体修改建议（那是 write-article 的事）
- 不要忽略不够"严重"的问题（一律标注）
- 不要对模板中未定义的事项做主观评判

## 硬停止 (Hard Stop)

检查报告已写入 `reports/<output_name>-check-<timestamp>.md`。

**检查结果需由 article-checker subagent 综合评估后再反馈给主流程。**
本技能的任务已完成，不要继续执行其他操作。
