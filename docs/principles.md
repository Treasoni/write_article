# 核心原则

> 本文档由 CLAUDE.md 拆分而来。做任何决策前，你【必须】调用 Read 工具读取本文档。这些原则来自实战经验，违反任一条都可能导致任务失败。

---

## 1. 三道防线防 Prompt Drift

多阶段任务中，模型容易在执行过程中"漂移"，忘记最初的约束。本项目设置三道防线：

| 防线 | 机制 | 说明 |
|------|------|------|
| 第一道 | Skill 否定约束 | 每个 Skill 和 Subagent 末尾有明确禁令（"严禁进入下一阶段"） |
| 第二道 | 状态框 | 你在每个阶段边界打印状态框，向用户确认当前进度 |
| 第三道 | Skill 级硬停止 | 每次调用 Skill 时重新读取刹车指令，阻止越界 |

**反面案例**：某次任务中 write-article 直接跳过了审核环节进入 Phase 4，因为缺少第一道防线的否定约束。

---

## 2. 绝不允许跳步

- Phase 0 → 1 → 2 → 3 → 4 必须严格按序执行
- 模板未审核通过 → **绝不能**开始写文章
- 资料未整理完 → **绝不能**开始写文章
- 文章未审核通过 → **绝不能**标记完成

**如果用户要求跳步**：解释原因并坚持流程。用户可能不了解阶段间的依赖关系，你的职责是守住流程。

---

## 3. 尊重审核结果

- template-checker 说不通过 → 必须修改后重新审核
- article-checker 说不通过 → 必须修改后重新审核
- **不要**试图说服审核员"这个其实没问题"
- 每个审核循环最多 **3 轮**，超过 3 轮请人工介入决策

**反面案例**：某次审核中模型花了 5 轮和 article-checker 争论一个标题格式问题，浪费了大量上下文。3 轮后直接询问用户即可。

---

## 4. 最小改动原则

- 资料整理时不修改 raw/ 下的原始文件
- 只更新 sources.md 的分类和标签
- 明显不适合的资料标注"未采用"而不是删除
- 文章修改时优先修改有问题的章节，而非整篇重写

---

## 5. 渐进式披露

CLAUDE.md 只是地图（~100 行），详细文档在 docs/ 目录。模型有"过度自信"倾向——如果不加强制约束，它会凭记忆脑补流程而不是读取文件。

**所有 docs/ 链接必须使用强指令格式：**
```
❌ 错误：参考 docs/workflow.md
✅ 正确：你【必须】调用 Read 工具读取 docs/workflow.md 中对应章节。严禁凭记忆执行。
```
